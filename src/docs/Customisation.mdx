# Customising Cast UI

You can change how Cast UI looks without editing any component code. This
guide covers every option, starting with the simplest.

**Quick reference:**

| You want to… | Do this |
|---|---|
| Use dark mode | Set `colorMode="dark"` on `ThemeProvider` |
| Change your brand colours | Pass a `colors` prop to `ThemeProvider` |
| Restyle from Figma, without code changes | Export a theme with the cast-sync plugin |
| Make spacing tighter or roomier | Set `density` on `ThemeProvider` |
| Build your own components on the same foundations | Use the `useTheme` hook and the exported tokens |
| Adjust one specific component | Pass it a `style` prop |

---

## First: wrap your app in ThemeProvider

All customisation happens through `ThemeProvider`. Wrap it around your app
once, and every Cast UI component inside picks up your settings
automatically.

```tsx
import { ThemeProvider } from '@castui/cast-ui';

export function App() {
  return (
    <ThemeProvider density="default">
      {/* your app here */}
    </ThemeProvider>
  );
}
```

> No `ThemeProvider`? Components still work — they use the default spacing,
> light mode, and the built-in colour palette.

---

## 1. Density — control how tight or roomy spacing feels

Cast UI has three density settings. Changing density scales the padding and
spacing of every component at the same time:

| Density | Best for | Spacing feel |
|---|---|---|
| `compact` | Dashboards, data-heavy screens, power users | Tight |
| `default` | Most interfaces | Balanced |
| `comfortable` | Touch-first apps, accessibility, onboarding | Spacious |

You can switch density while the app is running — no rebuild needed:

```tsx
const [density, setDensity] = useState<'compact' | 'default' | 'comfortable'>('default');

<ThemeProvider density={density}>
  <Button onPress={() => setDensity('compact')}>Go compact</Button>
</ThemeProvider>
```

**What changes:** padding, gaps, and spacing.
**What stays the same:** colours, corner rounding, typography, and icon sizes.

Because colours and type never change, your brand looks the same at every
density.

---

## 2. Colour mode — light and dark

Every component supports light and dark mode. Switch with the `colorMode`
prop. These are the same light and dark modes as the Figma file, so the two
always match.

To follow the user's system setting:

```tsx
import { useColorScheme } from 'react-native';

export function App() {
  const systemScheme = useColorScheme(); // reads the OS light/dark setting

  return (
    <ThemeProvider colorMode={systemScheme === 'dark' ? 'dark' : 'light'}>
      {/* your app here */}
    </ThemeProvider>
  );
}
```

Cast UI components don't control your screen's background — set it yourself
using the theme's surface colour, so components sit on the right canvas:

```tsx
import { useTheme } from '@castui/cast-ui';

function Screen({ children }) {
  const { scheme } = useTheme();
  return <View style={{ flex: 1, backgroundColor: scheme.surface.base }}>{children}</View>;
}
```

**What changes:** every colour — buttons, surfaces, text, form controls,
overlays.
**What stays the same:** spacing, typography, and corner rounding. Filled
(`bold`) brand and danger buttons also keep their light-mode colours in dark
mode, so primary actions stay recognisable.

## 3. Colour overrides — rebrand with one prop

To change Cast UI's colours, pass a `colors` prop to `ThemeProvider`. You
only need to write the colours you want to change — everything you leave out
keeps its default value.

A few terms used in the colour system:

- **Intent** — what a colour means: `neutral` (everyday), `brand` (your
  primary colour), or `danger` (destructive actions).
- **Prominence** — how visually heavy it is: `bold` (filled), `default`
  (outlined), or `subtle` (no background until hovered).
- **State** — `default`, `hover`, or `active` (pressed).
- Each combination has three values: `bg` (background), `fg` (text and
  icons), and `border`.

### Example: make the brand colour purple

```tsx
<ThemeProvider
  density="default"
  colors={{
    brand: {
      bold: {
        default: { bg: '#7C3AED', fg: '#FFFFFF', border: '#7C3AED' },
        hover:   { bg: '#6D28D9', fg: '#FFFFFF', border: '#6D28D9' },
        active:  { bg: '#5B21B6', fg: '#FFFFFF', border: '#5B21B6' },
      },
      default: {
        default: { bg: '#FFFFFF', fg: '#7C3AED', border: '#C4B5FD' },
        hover:   { bg: '#F5F3FF', fg: '#6D28D9', border: '#A78BFA' },
        active:  { bg: '#EDE9FE', fg: '#5B21B6', border: '#8B5CF6' },
      },
      subtle: {
        default: { bg: 'transparent', fg: '#7C3AED', border: 'transparent' },
        hover:   { bg: '#F5F3FF', fg: '#6D28D9', border: 'transparent' },
        active:  { bg: '#EDE9FE', fg: '#5B21B6', border: 'transparent' },
      },
    },
  }}
>
  <App />
</ThemeProvider>
```

Only `brand` changes here — `neutral` and `danger` keep their defaults.

### Or skip the hand-typing: export a theme from Figma

If you have your own copy of the cast-ui-kit Figma file, you don't
need to write any of the above by hand. The **cast-sync** Figma plugin (in
this repo's `cast-sync/` folder) reads the colour variables from Figma and
downloads them as a ready-made `cast-theme.json` file:

```tsx
import theme from './cast-theme.json';

<ThemeProvider colorMode="light" colors={theme.colors.light}>
  <App />
</ThemeProvider>
```

The workflow: recolour the variables in Figma → run the plugin → replace the
JSON file in your app. No code changes.

The plugin works with any Figma file whose variables follow the
`intent/{intent}/{prominence}/{state}/{bg|fg|border}` naming. See
`cast-sync/README.md` for setup.

### The full colour structure

Each intent (`brand`, `neutral`, `danger`) has three prominences, and each
prominence has three states:

```text
intent
  ├── default (outlined)
  │   ├── default  → bg, fg, border
  │   ├── hover    → bg, fg, border
  │   └── active   → bg, fg, border
  ├── bold (filled)
  │   └── ...
  └── subtle (ghost)
      └── ...
```

## 4. Use the tokens directly — build your own components

Everything Cast UI uses internally is exported, so your own components can
share the same colours, spacing, and typography. The `useTheme` hook gives
you the live theme — it updates automatically when density or colour mode
changes:

```tsx
import { useTheme, intentColors, fontFamily, fontWeight, label } from '@castui/cast-ui';

function CustomCard({ children }) {
  const { components } = useTheme();
  // Read the current density's button tokens (or any other component's tokens)
  const padding = components.button.default.paddingX;

  return (
    <View style={{ padding, borderRadius: 8 }}>
      {children}
    </View>
  );
}
```

### What's exported

| Export | What it gives you |
|---|---|
| `lightColors` / `darkColors` | The complete colour set for each mode — intents, surfaces, text, form controls |
| `intentColors` | Light-mode intent colours: intent → prominence → state → {bg, fg, border} |
| `disabledColors` | Light-mode disabled colours: {bg, fg, border} |
| `controlTokens` | Shared constants, e.g. border width |
| `fontFamily` | Font families per platform (sans, mono, serif) |
| `fontWeight` | Named font weights (light through bold) |
| `label`, `title`, `body`, `caption` | Typography scales for each text role |
| `themes` | The raw spacing values for all three densities |

## 5. How the token system works

Cast UI's values come from three layers of **design tokens** — named values
shared between Figma and code. Knowing the layers helps you pick the right
place to customise:

```
┌──────────────────────────────────────────────────────┐
│  Component tokens                                     │
│  e.g. button.default.paddingX = 14                    │
│  → "How much padding does a button have?"             │
│  → This layer changes with density                    │
├──────────────────────────────────────────────────────┤
│  Semantic tokens                                      │
│  e.g. intent/brand/bold/default/bg = #2563EB          │
│  → "What colour is a primary action?"                 │
│  → This layer changes with colour mode, and is what   │
│    the `colors` prop overrides                        │
├──────────────────────────────────────────────────────┤
│  Primitive tokens                                     │
│  e.g. colours/blue/600 = #2563EB, space/8 = 14        │
│  → "What exactly is blue-600?"                        │
│  → The raw palette — you rarely touch this directly   │
└──────────────────────────────────────────────────────┘
```

**Rule of thumb:** customise at the highest layer that does the job.

- Want dark mode? → Set `colorMode="dark"`.
- Want a different brand colour? → Override `colors.brand`.
- Want tighter spacing? → Set `density="compact"`.
- Want to tweak one specific button? → Give it a `style` prop.

## 6. Icons — bring your own

Cast UI doesn't lock you into an icon library. The `leadingIcon` and
`trailingIcon` props accept any React element, so use whatever you prefer:

```tsx
import { Ionicons } from '@expo/vector-icons';
import { Button } from '@castui/cast-ui';

<Button
  intent="brand"
  prominence="bold"
  leadingIcon={<Ionicons name="add" size={16} color="#FFFFFF" />}
>
  Add item
</Button>
```

Size icons at **16 × 16** to match the design spec.

## 7. Nested ThemeProviders — different settings in different places

You can put one `ThemeProvider` inside another. The inner one wins for the
components inside it — this works for `density` and `colorMode` alike:

```tsx
<ThemeProvider density="comfortable">
  {/* Main app — comfortable spacing */}
  <Header />

  <ThemeProvider density="compact">
    {/* Data table — compact spacing to fit more rows */}
    <DataTable />
  </ThemeProvider>
</ThemeProvider>
```

This lets you mix settings on one screen — for example, spacious navigation
next to a dense data grid.
