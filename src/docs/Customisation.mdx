# Customising Cast UI

Cast UI is built on a **3-layer design token system** designed for
straightforward customisation вҖ” from quick brand changes to full white-label
deployments.

---

## Quick start

Wrap your app with `ThemeProvider`. Every Cast UI component inside will pick up
the theme automatically.

```tsx
import { ThemeProvider } from '@castui/cast-ui';

export function App() {
  return (
    <ThemeProvider density="default">
      {/* your app here */}
    </ThemeProvider>
  );
}
```

> If you omit the `ThemeProvider`, components fall back to the **default**
> density with the built-in colour palette.

---

## 1. Density themes вҖ” spacing that adapts to context

Cast UI ships with three density modes that scale padding, gap, and spacing
across every component simultaneously:

| Density | Best for | Padding feel |
|---|---|---|
| `compact` | Data-dense screens, dashboards, power users | Tight |
| `default` | General-purpose interfaces | Balanced |
| `comfortable` | Accessibility, touch-first, onboarding flows | Spacious |

**Switch at runtime** вҖ” no rebuild needed:

```tsx
const [density, setDensity] = useState<'compact' | 'default' | 'comfortable'>('default');

<ThemeProvider density={density}>
  <Button onPress={() => setDensity('compact')}>Go compact</Button>
</ThemeProvider>
```

**What changes:** padding, gap, spacing tokens.
**What stays constant:** colours, border-radius, focus rings, typography, icon sizes.

This means your brand identity is preserved regardless of which density a user selects.

---

## 2. Colour overrides вҖ” rebrand in one prop

Pass a `colors` prop to `ThemeProvider` to override any intent colour at any
depth. The override is **deep-merged** вҖ” only provide what you want to change.

### Example: purple brand

```tsx
<ThemeProvider
  density="default"
  colors={{
    brand: {
      bold: {
        default: { bg: '#7C3AED', fg: '#FFFFFF', border: '#7C3AED' },
        hover:   { bg: '#6D28D9', fg: '#FFFFFF', border: '#6D28D9' },
        active:  { bg: '#5B21B6', fg: '#FFFFFF', border: '#5B21B6' },
      },
      default: {
        default: { bg: '#FFFFFF', fg: '#7C3AED', border: '#C4B5FD' },
        hover:   { bg: '#F5F3FF', fg: '#6D28D9', border: '#A78BFA' },
        active:  { bg: '#EDE9FE', fg: '#5B21B6', border: '#8B5CF6' },
      },
      subtle: {
        default: { bg: 'transparent', fg: '#7C3AED', border: 'transparent' },
        hover:   { bg: '#F5F3FF', fg: '#6D28D9', border: 'transparent' },
        active:  { bg: '#EDE9FE', fg: '#5B21B6', border: 'transparent' },
      },
      ringColour: '#A78BFA',
    },
  }}
>
  <App />
</ThemeProvider>
```

Only the `brand` intent changes вҖ” `neutral` and `danger` stay untouched.

### Colour structure

Each intent (`brand`, `neutral`, `danger`) has three prominences, each with
three states:

```text
intent
  в”ңв”Җв”Җ default (outlined)
  в”Ӯ   в”ңв”Җв”Җ default  вҶ’ bg, fg, border
  в”Ӯ   в”ңв”Җв”Җ hover    вҶ’ bg, fg, border
  в”Ӯ   в””в”Җв”Җ active   вҶ’ bg, fg, border
  в”ңв”Җв”Җ bold (filled)
  в”Ӯ   в””в”Җв”Җ ...
  в”ңв”Җв”Җ subtle (ghost)
  в”Ӯ   в””в”Җв”Җ ...
  в””в”Җв”Җ ringColour   вҶ’ focus ring colour
```

## 3. Using tokens directly вҖ” build custom components on the same system

All tokens and the `useTheme` hook are exported, so you can build custom
components that share the same token foundation:

```tsx
import { useTheme, intentColors, fontFamily, fontWeight, label } from '@castui/cast-ui';

function CustomCard({ children }) {
  const { components } = useTheme();
  // Access the current density's button tokens (or any future component tokens)
  const padding = components.button.default.paddingX;

  return (
    <View style={{ padding, borderRadius: 8 }}>
      {children}
    </View>
  );
}
```

### Available token exports

| Export | What it contains |
|---|---|
| `intentColors` | Full colour map: intent вҶ’ prominence вҶ’ state вҶ’ {bg, fg, border} |
| `disabledColors` | Shared disabled state: {bg, fg, border} |
| `controlTokens` | Shared constants: borderWidth |
| `fontFamily` | Platform-aware font families (sans, mono, serif) |
| `fontWeight` | Named weights (light through bold) |
| `label` | Label typography scales: sm, md, lg |
| `themes` | All three density definitions as raw objects |

```
в”Ӯ  вҶ’ "How much padding does a default button have?"     в”Ӯ

в”Ӯ  вҶ’ "What colour is a primary action?"                 в”Ӯ
в”Ӯ  вҶ’ Override via ThemeProvider colors prop              в”Ӯ
в”ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Ө

```

**Rule of thumb:** customise at the highest layer that achieves your goal.
- Want to change brand colour? вҶ’ Override `colors.brand` on ThemeProvider.
- Want tighter spacing? вҶ’ Switch `density` to `compact`.
- Want to change a single button's padding? вҶ’ Pass a `style` prop.

## 5. Icons вҖ” bring your own

Cast UI is icon-agnostic. The `leadingIcon` and `trailingIcon` props accept
any `React.ReactNode`. Use whichever icon library you prefer:

```tsx
import { Ionicons } from '@expo/vector-icons';
import { Button } from '@castui/cast-ui';

<Button
  intent="brand"
  prominence="bold"
  leadingIcon={<Ionicons name="add" size={16} color="#FFFFFF" />}
>
  Add item
</Button>
```

Icons should be **16 Г— 16** to match the Figma spec.

---

## 6. Nested ThemeProviders вҖ” different zones, different densities

ThemeProviders can be nested. Inner providers override outer ones:

```tsx
<ThemeProvider density="comfortable">
  {/* Main app вҖ” comfortable spacing */}
  <Header />

  <ThemeProvider density="compact">
    {/* Data table вҖ” compact spacing for information density */}
    <DataTable />
  </ThemeProvider>
</ThemeProvider>
```

This lets you mix densities in a single screen вҖ” spacious navigation alongside
a dense data grid, for example.
